# Documentation Standard

## Objective

Define the minimum structure and quality criteria for documentation stored in this repository.

## General rules

- Use clear and descriptive titles
- Prefer operational language over theoretical language
- Keep sections short and actionable
- Update outdated content when modifying a related document

## Required structure for runbooks

- Purpose
- When to use
- Initial checks
- Steps
- Validation
- Escalation

## Required structure for postmortems

- Summary
- Impact
- Root cause
- Resolution
- Follow-up actions

## Review criteria

A document is considered ready when:

- it follows the expected structure
- it is understandable without extra context
- it avoids sensitive information
- it adds operational value
