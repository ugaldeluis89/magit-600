# GitHub como KMS para un equipo SRE

> **Curso:** MAGIT-600 – Sistemas para la Gestión de la Información Empresarial  
> **Programa:** Maestría Profesional en Gestión de la Información y Tecnología  
> **Universidad:** Universidad Fidélitas  
> **Estudiante:** Luis Andrés Ugalde Mena

Este sitio corresponde a un demo académico orientado a mostrar cómo **GitHub puede utilizarse como plataforma de gestión del conocimiento** para un equipo de Site Reliability Engineering (SRE).

---

## Propósito del demo

El objetivo es evidenciar cómo un repositorio bien estructurado puede apoyar la operación diaria de un equipo SRE mediante documentación versionada, revisión colaborativa y validaciones automáticas.

En este contexto, GitHub no se utiliza únicamente como repositorio de código, sino como un entorno para:

- documentar conocimiento operativo
- capturar lecciones aprendidas
- establecer estándares de documentación
- mantener trazabilidad de cambios
- reforzar consistencia mediante automatización

---

## Relación con el curso

Este demo forma parte del trabajo académico desarrollado en el curso **MAGIT-600**, enfocado en analizar estrategias y herramientas para la gestión de la información empresarial. El caso se centra en la gestión del conocimiento dentro de un equipo SRE asociado a una operación tecnológica compleja, donde la disponibilidad, la consistencia y la reutilización del conocimiento resultan críticas para el soporte del negocio.

## Flujo que demuestra este repositorio

1. Se identifica una necesidad, incidente o mejora.
2. Se documenta mediante issue o template.
3. Se propone un cambio mediante pull request.
4. Se ejecutan validaciones automáticas.
5. Se integra el conocimiento aprobado.
6. El contenido queda disponible para reutilización operativa.

---

## Valor esperado

La propuesta busca demostrar que una gestión del conocimiento más estructurada puede contribuir a:

- reducir la dependencia de conocimiento tácito
- mejorar la consistencia operativa
- facilitar la transferencia de contexto entre zonas horarias
- disminuir reprocesos
- fortalecer la continuidad del servicio
