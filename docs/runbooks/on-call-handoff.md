# Runbook: On-Call Handoff

## Purpose

Provide a consistent format for transferring operational context between on-call shifts.

## Required handoff content

- active incidents
- current severity and impact
- pending actions
- temporary mitigations in place
- risks for the next shift
- relevant links

## Handoff checklist

1. Confirm the current status of all active incidents.
2. Summarize actions completed during the shift.
3. List pending actions and clear next steps.
4. Highlight any unresolved risk or customer-facing impact.
5. Share links to dashboards, tickets, pull requests, or runbooks.

## Expected outcome

The receiving engineer should be able to continue the shift without reconstructing context from multiple sources.
