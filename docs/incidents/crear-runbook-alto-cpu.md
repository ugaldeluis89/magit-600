# Crear runbook para incidente de alto CPU

## Contexto

Se ha identificado la necesidad de estandarizar la respuesta ante incidentes de alto uso de CPU.

## Problema

Actualmente no existe un procedimiento documentado consistente.

## Propuesta

Desarrollar un runbook con pasos claros de diagnóstico, contención y validación.

## Resultado esperado

Un runbook reutilizable que mejore la consistencia operativa.
